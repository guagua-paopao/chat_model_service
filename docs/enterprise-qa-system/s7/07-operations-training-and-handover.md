# 运营培训与移交

## 培训矩阵

| 受众 | 必会内容 | 演练证据 |
|---|---|---|
| 用户 | 高质量提问、核对引用、识别拒答、反馈 | UC-01～05 |
| 知识管理员 | 分类/ACL、摄取、预览、发布、回滚、失效处理 | 合成文档演练 |
| 开发 | 本地环境、模块/API、迁移、契约、PR 与事故调试 | check + trace |
| SRE | SLO、告警、容量、Provider 故障、灰度、回滚、恢复 | stop/rollback drill |
| 安全/审计 | IAM、事件、证据保全、哈希验证、数据请求 | audit/integrity drill |

## 移交清单

- Product Owner、Business Owner、Data Owner、Security Owner、Service Owner、DBA、on-call 主备和供应商联系人具名。
- 服务目录、架构图、数据分类、接口、迁移、配置、仪表盘、告警、Runbook、DR、风险接受与预算链接齐全。
- 镜像 digest、SBOM、签名/provenance、评测 run、UAT、审批、部署/回滚 ID 可相互追踪。
- 值班权限、Pager route、升级路径、供应商工单和密钥轮换已实际验证。
- 发布后观察期、复盘日期、KPI 周报/月报、知识新鲜度和成本对账 Owner 明确。

## 运营节奏

- 每日：错误预算、安全事件、队列、Provider 与异常成本。
- 每周：负反馈/拒答/引用、知识过期、Top 无答案、风险 backlog。
- 每月：SLO、账单对账、访问复核、漏洞/SBOM、备份恢复抽查。
- 每季度：DR、红队、权限再认证、供应商与模型路线复评。

本地项目提供角色、模板和演练路径，不填写虚构姓名、电话、预算或生产链接。
