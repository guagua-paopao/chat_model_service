# 00. 项目章程与范围

## 1. 项目概述

### 1.1 业务问题

企业知识通常分散在制度文档、产品手册、工单、FAQ 与内部系统中。传统关键词搜索难以理解问题意图，通用大模型又无法保证回答来自企业认可资料。项目通过“检索增强生成（RAG）+ 权限过滤 + 引用溯源”缩短查找时间，同时保留可验证性与治理能力。

### 1.2 产品愿景

让授权用户通过自然语言，在 30 秒内获得基于企业受控知识、带来源引用、可追踪且能安全拒答的答案；让内容管理员能持续发布、回滚和评估知识，让运维人员能量化质量、成本和风险。

### 1.3 项目目标

| ID | 目标 | 度量方式 | 首次生产门槛 |
|---|---|---|---|
| O-01 | 提升知识获取效率 | 任务完成时长中位数 | 相比基线下降 ≥ 50% |
| O-02 | 保证有据可查 | 需要事实依据的回答含有效引用比例 | ≥ 95% |
| O-03 | 控制幻觉 | 黄金集忠实度/groundedness | ≥ 0.90 |
| O-04 | 正确拒答 | 无依据或越权问题的拒答准确率 | ≥ 90% |
| O-05 | 可稳定运行 | 月度 API 可用性 | ≥ 99.9% |
| O-06 | 可管理成本 | 每个答案的模型与检索成本 | 可归集、可设预算、可告警 |
| O-07 | 权限不降级 | 跨租户/越权检索测试 | 0 个成功越权用例 |

> 指标门槛是立项基线。项目第 0 阶段须用真实业务样本校准，不得用演示数据直接承诺业务收益。

## 2. 范围边界

### 2.1 MVP（第 1–10 周，必须完成）

- Web 端登录、对话列表、流式回答、停止生成、重试与反馈。
- 兼容至少两个模型提供方的 Chat 与 Embedding 接入，具备超时、重试、熔断、限流与成本记录。
- 知识库、PDF/DOCX/TXT/Markdown 上传、异步解析、切分、向量化、版本与失败重试。
- 基于租户和文档 ACL 的混合检索、重排、引用、无依据拒答。
- 会话与消息持久化；管理员查看任务、失败原因、基础使用量和审计日志。
- OpenAPI、数据库迁移、单元/集成/E2E、黄金集评测、Docker Compose 本地环境。

### 2.2 Production Ready（第 11–16 周，必须完成）

- 企业 OIDC/SSO、RBAC、限额、密钥托管、敏感信息处理和审计导出。
- Kubernetes/Helm、开发/测试/预生产/生产四环境、CI/CD、镜像与依赖扫描。
- OpenTelemetry 全链路、仪表盘、SLO 告警、成本与配额治理。
- 压测、故障注入、备份恢复演练、灰度与一键回滚。
- UAT、运行手册、值班与事件响应、上线评审、项目移交。

### 2.3 第二阶段候选（不阻塞首发）

- 企业微信/飞书/钉钉、客服系统或浏览器插件渠道。
- Confluence、SharePoint、网页、数据库等增量连接器。
- OCR、表格和复杂版面专项优化；图片、多模态问答。
- Agent 工具调用、业务流程写操作、人工审批工作流。
- 多语言检索、个性化推荐、知识图谱、模型微调。
- 独立向量数据库、跨地域双活、计费结算。

### 2.4 明确不在首期范围

- 训练基础大模型或从零训练 Embedding 模型。
- 无人审批的高风险业务写操作，例如转账、删库、自动发外部邮件。
- 将模型输出作为医疗、法律、财务等高风险决策的唯一依据。
- 承诺回答 100% 正确；产品契约是“证据、质量度量与安全降级”，不是消灭概率性。
- 对开源项目进行换皮交付，或在未完成许可证审查前复制受限制代码。

## 3. 目标用户和核心场景

| 用户 | 主要任务 | 价值 | 首期示例 |
|---|---|---|---|
| 普通员工 | 查询制度、产品、流程 | 减少搜索和询问专家时间 | “差旅报销需要哪些材料？” |
| 客服/一线支持 | 快速定位标准答案 | 提升首解率与一致性 | “设备错误码 E104 如何排查？” |
| 知识管理员 | 上传、发布、回滚、观察失败 | 保证内容新鲜和可控 | 发布新版售后政策 |
| 业务管理员 | 配置模型、提示词、配额、质量集 | 按场景治理效果和成本 | 财务知识库使用更严格模型 |
| 安全审计员 | 查询登录、知识与问答审计 | 发现滥用并满足审计要求 | 谁访问过受限制度 |
| 平台运维 | 监控容量、成本、延迟、错误 | 保障 SLO | 模型供应商异常时切换 |
| 第三方系统 | 通过 API 发起问答/检索 | 嵌入业务流程 | 工单系统推荐知识文章 |

## 4. 非功能基线

| 类别 | 基线要求 | 验证方式 |
|---|---|---|
| 可用性 | 生产 API 月度 99.9%；计划维护另行约定 | SLI 仪表盘 |
| 延迟 | 缓存未命中且上游正常时，首 token P95 ≤ 2.5s；完整回答 P95 ≤ 15s | 预生产压测+生产 RUM |
| 吞吐 | 基线 50 RPS 查询、200 个并发 SSE；可水平扩展 | k6/Locust |
| 摄取 | 50 MB 以内普通文档 P95 在 10 分钟内可检索 | 任务指标 |
| 数据 | 默认 RPO ≤ 15 分钟、RTO ≤ 60 分钟 | 恢复演练 |
| 安全 | TLS、OIDC、最小权限、租户隔离、静态/传输加密、审计 | 安全评审/渗透测试 |
| 兼容 | 最近两个主版本 Chrome/Edge；API 向后兼容 | 浏览器矩阵/契约测试 |
| 可维护 | 关键模块单测覆盖率 ≥ 80%，高风险分支 100% 场景覆盖 | CI 报告 |
| 可观测 | 每次问答可通过 `request_id/trace_id` 关联检索、模型、成本与错误 | Trace 抽检 |
| 可访问性 | 核心 Web 流程以 WCAG 2.1 AA 为设计目标 | 自动+人工检查 |

延迟 SLO 应区分“平台自身耗时”和“外部模型耗时”，但用户体验 SLI 必须统计端到端时间，不能通过排除上游掩盖真实体验。

### 6.2 RACI 摘要

| 工作项 | 产品 | 技术负责人 | 研发 | 测试 | SRE | 安全/法务 |
|---|---|---|---|---|---|---|
| 范围与优先级 | A/R | C | C | C | I | C |
| 架构/API/数据 | C | A/R | R | C | C | C |
| 黄金集与质量门槛 | A/R | C | C | R | I | C |
| 威胁模型与隐私 | C | R | R | C | C | A |
| 生产发布 | C | A | R | R | R | C |
| 事故指挥 | I | C | C | C | A/R | C |

R=执行，A=最终负责，C=会签，I=知会。

## 7. 预算与容量估算方法

立项阶段不要先填写一个固定的“每月模型费”，而应使用可追溯公式：

```text
月输入 token = 月问题数 × (系统提示 + 对话历史 + 检索上下文 + 用户问题)
月输出 token = 月问题数 × 平均回答 token
模型月成本 = Σ(输入 token × 输入单价 + 输出 token × 输出单价)
Embedding 成本 = 新增/变更文档 token × embedding 单价
平台成本 = API/Worker/K8s + PostgreSQL + Redis + 对象存储 + 监控日志 + 出网
总预算 = 模型 + 平台 + 安全工具 + 人力 + 20% 风险缓冲
```

必须用业务样本测量平均 token；分别估算基准、2 倍峰值和 5 倍增长情景。设置租户日/月预算、单请求 token 上限、上下文裁剪与异常突增告警。

| ID | 风险 | 概率/影响 | 预防与缓解 | Owner | 触发器 |

| R-04 | 模型供应商故障/限流 | 中/高 | 超时、退避、熔断、备用模型、降级文案 | 后端/SRE | 5xx/429 激增 |

| R-06 | 成本失控 | 中/高 | 配额、路由、缓存、token 上限、预算告警 | 产品/SRE | 日成本 > 预算 120% |

## 9. 变更控制

任何新增模型、数据源、外部渠道、工具写操作或新的敏感数据类别，都必须提交变更单，至少包含：业务价值、数据流变化、威胁模型差异、接口/迁移、成本、测试、回滚和许可证影响。

变更分级：

- **标准变更**：不改变外部契约且有自动回滚，技术负责人批准。
- **重要变更**：改变 API、数据结构、模型策略或 SLO，需产品、架构、测试、SRE 会签。
- **高风险变更**：新敏感数据、外部写操作、跨境/跨域数据流或权限模型变化，必须增加安全、数据负责人和法务会签。

## 10. 阶段 0 必须确认的开放项

1. 目标是内部员工、外部客户还是两者；外部场景对隔离、滥用防护和 SLA 要求更高。
2. 首批知识来源、文档类型、规模、更新频率、密级和原系统权限如何映射。
3. 可使用的云模型/私有模型、数据是否允许离开指定网络边界。
4. 企业身份源、组织架构、用户离职和权限回收时效。
5. 峰值并发、用户地域、可用性窗口、RPO/RTO 与预算上限。
6. 审计和对话保留期限，以及用户删除/导出流程。
7. 哪些回答必须人工复核，哪些场景必须拒答或转人工。
