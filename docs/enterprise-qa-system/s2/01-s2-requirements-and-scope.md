# S2 需求、应用场景与范围

## 1. 阶段目标

在不接触真实企业知识的前提下，完成“认证用户提问 → 模型路由 → 流式响应 → 取消/重试 → 用量和成本落账”的纵向切片，并把供应商差异隔离在 Adapter 内。S2 是后续 RAG 的生成底座，不是知识问答验收版本。

## 2. 角色与应用场景

| 场景 | 参与者 | 主流程 | 验收结果 |
|---|---|---|---|
| UC-S2-01 通用问答 | 已认证员工 | 创建会话，发送普通问题，接收 SSE 增量 | 事件顺序稳定，最终消息和用量入库 |
| UC-S2-02 停止生成 | 已认证员工 | 生成中点击停止 | 同租户同用户可取消；最终状态为 `cancelled` |
| UC-S2-03 失败重试 | 已认证员工 | 上游失败后对助手消息重试 | 复用原用户消息，新建助手消息，保留旧失败证据 |
| UC-S2-04 Provider 故障 | 平台/SRE | 主路由 429/超时 | 首个可见 token 前按策略重试/切备用；错误被归一化 |
| UC-S2-05 成本审计 | FinOps/审计 | 按 request/message 查询调用记录 | Provider、模型、token、价格快照、金额可追踪 |
| UC-S2-06 非法知识模式 | 用户/测试 | 请求 grounded/search 或提交 KB ID | 409 明确拒绝，不伪造引用或知识能力 |

## 3. 功能需求

| ID | 优先级 | 需求 | 验收标准 |
|---|---|---|---|
| FR-S2-001 | Must | 提供 Provider-neutral `ModelAdapter` | 领域/聊天层不导入任何厂商 SDK；Fake 与 OpenAI-compatible 契约测试通过 |
| FR-S2-002 | Must | 同步与 SSE 两种回答 | `stream=false` 返回完整对象；`stream=true` 返回约定事件序列 |
| FR-S2-003 | Must | 路由、重试、备用与熔断 | 429/超时等可重试错误在预算内处理；不可重试错误立即终止 |
| FR-S2-004 | Must | 不混合两个模型的可见输出 | 一旦发送 `message.delta`，该次调用失败时不得切备用继续拼接 |
| FR-S2-005 | Must | 取消和断线清理 | 主动取消幂等；客户端断开释放并发许可并落 `cancelled` |
| FR-S2-006 | Must | 用量/成本账本 | 成功回答恰有一条账本记录；上游无 usage 时标记 `estimated=true` |
| FR-S2-007 | Must | 调用尝试审计 | 每次 route attempt 独立记录状态、耗时、安全错误码，不保存密钥 |
| FR-S2-008 | Must | 租户/用户作用域 | 取消、重试、读取消息均强制可信 `tenant_id` 和私有会话 `user_id` |
| FR-S2-009 | Must | 输入/速率/并发配额 | 超限使用稳定 Problem code，且不创建悬空消息 |
| FR-S2-010 | Must | 模型发现 | `/models` 只返回公共能力/策略，不暴露 endpoint、密钥或供应商内部配置 |
| FR-S2-011 | Must | 开发故障注入 | Fake Adapter 可确定性触发 429、全失败、超时、中断、缺 usage 和慢响应 |
| FR-S2-012 | Should | Web 完整体验 | 登录后可新建会话、流式显示、停止、重试、查看 token/成本 |

## 4. 非功能需求

- 安全：Provider 密钥不得出现在 API 响应、客户端 bundle、数据库、普通日志和 Git；生产禁用 Fake。
- 可用性：独立设置连接、首 token、总时限；重试有最大次数、指数退避和抖动；并发有上限。
- 一致性：用户消息和助手占位消息同事务创建；完成消息、用量账本和审计同事务提交。
- 可追踪：`request_id` 贯穿消息、调用尝试、SSE 和用量；`trace_id` 返回于完成事件。
- 兼容：公开请求不含厂商特有参数；新增 Adapter 不改变聊天业务接口。
- 性能目标：目标环境 TTFT P95 ≤ 2.5 秒、完整回答 P95 ≤ 15 秒；S2 Fake 功能验证不等于真实模型性能证据。
- 可维护：错误只有安全文案对外；原始上游响应不可原样写日志或返回。

## 5. 范围外

- 文档上传、解析、切分、Embedding、混合检索、重排、引用和有依据拒答（S3/S4）。
- 真实模型的效果、区域、配额、价格和数据处理条款验收。
- 跨设备 token 级流续传；S2 只支持读取最终/失败状态后重试。
- 工具调用、Agent 写操作、多模态和供应商专有高级参数。
- 分布式配额/取消、生产压测、灾备和 99.9% SLO 证明。

## 6. Definition of Done

代码、迁移、OpenAPI、Web、部署参数、单元/集成/契约测试、全栈 smoke、ADR、决策日志、风险和 Gate 必须同时更新。真实 Provider 未获批或未验证时，Gate 只能允许继续合成阶段开发，不能批准真实数据或生产。
