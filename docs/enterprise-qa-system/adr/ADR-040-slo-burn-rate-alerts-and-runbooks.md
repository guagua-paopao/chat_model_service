# ADR-040：SLO 告警采用多窗口燃尽率并绑定 Owner 与 Runbook

- 状态：Accepted
- 日期：2026-07-16
- Owner：SRE

## 决策

- 可用性目标沿用 99.9% 假设基线；严重错误使用 5 分钟和 1 小时组合的 14.4 倍燃尽率告警。
- API P95 持续 10 分钟超过 2.5 秒触发 P2；真实 TTFT 和完整回答 SLI 在目标 Provider 接入后分别计算。
- 遥测缺失、质量门禁失败、可用性燃尽和延迟均配置告警；每条规则必须包含 `owner`、阈值理由和 `runbook_url`。
- 告警规则先通过 `promtool`，再在目标环境做实际触发、路由、静默和恢复演练。

## 后果

本地具备可执行告警骨架；未验证生产 Alertmanager、Pager、值班轮转和真实流量，因此生产 Gate 仍未关闭。
