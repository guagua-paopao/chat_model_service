# ADR-035：S5 使用数据库串行化共享配额与并发租约

- 状态：Accepted
- 日期：2026-07-16
- Owner：Platform / API / SRE

## 背景

S2 内存速率计数和信号量不能跨实例。Redis Lua 可以提高吞吐，但会引入计数、账本、策略数据库三方一致性和额外故障模式；S5 首先需要一条可验证的强一致基线。

## 决策

- 在租户行事务锁下读取版本化 `quota_policies`，清理过期租约，检查 tenant/user 活动租约、分钟窗口、日 token 和月成本。
- 获取成功时同事务增加 `quota_windows` 并写带 TTL 和输入 token 预留量的 `quota_leases`；日限额以已结算账本、全部活动预留和当前估算之和检查。请求终态删除租约，进程崩溃由 TTL 恢复。
- 计费事实仍以追加式 `usage_ledger` 为准，窗口只用于准入，不作为账单。
- 任何数据库错误均 fail closed，不允许每个实例退回独立内存计数。
- Redis 可在压测证明数据库热点后作为实现替换，但必须保持原子语义和故障测试。

## 后果

速率和并发在多 API 实例间共享，运维依赖更少；高峰时 tenant 行可能成为热点。ADR-018 的“共享配额”部分得到实现，但跨实例取消信号仍未完成，因此生产多副本 Gate 仍未关闭。
