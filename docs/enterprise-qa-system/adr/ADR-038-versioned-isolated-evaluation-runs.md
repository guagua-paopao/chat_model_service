# ADR-038：使用版本化、租户隔离的评测运行作为发布证据

- 状态：Accepted
- 日期：2026-07-16
- Owner：AI Quality / Platform / Security

## 背景

S5 的单配置结构检查不能比较候选版本、基线回归、代码版本和数据集版本，也不适合作为长期发布证据。评测原始问答可能含敏感信息，不能无边界复制到治理库或指标标签。

## 决策

- 每次运行固定 `dataset_version_id`、SHA-256、候选配置完整快照、代码修订、评测器版本、阈值、基线与差异。
- 所有查询强制使用认证身份中的 `tenant_id`；客户端不能提交租户字段。
- 运行记录只保存候选指标和安全的 `case_id/control/check_code`，不保存原始问题、答案或文档正文。
- 本地同步评测器只允许 `local/test/dev`；staging/production 启动配置必须关闭它，并由批准的外部 Worker 执行同一契约。
- 任一安全控制未达标或质量分相对基线下降超过 2 个百分点时，门禁失败。

## 后果

评测可复现、可审计、可比较，失败可直接阻断发布。S6 的 24 条合成控制用例只是工程验证，不构成真实业务质量签字。
