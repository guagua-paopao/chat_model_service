# 12. 交付物与评审清单

## 1. Definition of Ready（进入开发前）

- [ ] 需求有唯一 ID、业务 Owner、用户/场景和 MoSCoW 优先级。
- [ ] 验收条件包含正常、失败、权限、幂等/重试和可观测性。
- [ ] 外部接口、字段、错误、兼容性和示例已评审；OpenAPI 草案已更新。
- [ ] 数据实体、分类、保留/删除、迁移与回滚影响已识别。
- [ ] 租户/ACL、Prompt 注入、成本/资源消耗和供应链影响已检查。
- [ ] 依赖、前置配置、测试数据/黄金样例和环境已准备。
- [ ] 工作量可在一个 Sprint 内完成；更大项已拆分为可独立验收纵切。

## 2. Definition of Done（故事完成）

- [ ] 代码经过格式、lint、类型、单元、集成、契约和必要 E2E。
- [ ] 正向/负向权限与跨租户测试通过；无 secret/敏感正文进入代码和日志。
- [ ] OpenAPI、字段字典、迁移、ADR/运行文档与 UI 文案同步。
- [ ] 新指标/日志/trace/告警可验证且低基数、符合隐私白名单。
- [ ] 失败、超时、重试、取消、幂等、降级和回滚路径已演示。
- [ ] 依赖/镜像/许可/SBOM 扫描通过；高风险项会签。
- [ ] 部署到 dev/test 并由产品/测试按验收条件接受。

## 3. 设计评审模板

```text
标题 / Owner / 状态 / 日期 / 评审人
背景与问题；目标/非目标；用户与场景
需求与成功指标；容量/成本假设
方案与数据流；API/事件；数据模型/迁移
身份/租户/ACL；数据分类/保留；威胁模型
可靠性/幂等/一致性；性能/扩缩；可观测性
测试与 AI 评测；发布/灰度/回滚/DR
替代方案与取舍；开放问题；ADR/批准
```

评审结论只允许 `approved / approved-with-actions / revise / rejected`；action 必须有 Owner 和截止时间。

## 4. PR 检查清单

### 作者

- [ ] 关联需求/设计/ADR，说明用户可见变化和不在范围内容。
- [ ] 标出 API、DB、Prompt/检索、权限、secret、成本和运维影响。
- [ ] 提供测试命令/结果、截图或 trace（不含敏感数据）。
- [ ] 迁移说明包含兼容、锁/数据量、校验和回滚/前向修复。
- [ ] 无不相关格式化或大范围重构；生成/第三方代码 provenance 清楚。

### 评审人

- [ ] 契约和错误语义稳定，客户端不能伪造租户/角色/内部字段。
- [ ] 所有数据访问 tenant-scoped，ACL 在召回前生效，缓存键安全。
- [ ] 重试有界且只针对瞬时错误，写操作幂等，状态机无竞态。
- [ ] 日志/错误/trace 不泄露；输出被正确转义/校验。
- [ ] 测试能在修复前失败，覆盖主要负向场景；指标可诊断真实故障。

## 5. API 评审清单

- [ ] 资源/方法/状态码、认证 scope、租户解析、分页、幂等、并发控制一致。
- [ ] 请求/响应字段含类型、必填、长度、枚举、默认、示例和敏感级别。
- [ ] 4xx/5xx 与 SSE 流后错误语义明确，错误码稳定且不泄露内部细节。
- [ ] 取消/超时/断线/重试/重复请求的最终状态明确。
- [ ] 兼容性、弃用、SDK/消费者契约和 rate/quota header 已考虑。
- [ ] 引用/下载/导出 URL 短期且访问时重新鉴权。

## 6. 数据库迁移检查清单

- [ ] expand/contract 顺序支持旧/新应用并存。
- [ ] 大表/索引/回填的锁、磁盘、WAL/复制和耗时经过估算/演练。
- [ ] 所有租户表有 tenant 约束与索引；外键归属不允许跨租户。
- [ ] 数据校验、回填断点续跑、失败告警和停止条件存在。
- [ ] 回滚可行；不可逆时有快照、前向修复和批准的风险窗口。
- [ ] 生产应用账号无 DDL/owner 权限。

## 7. Prompt/检索/模型配置发布清单

- [ ] 配置是不可变候选版本，有 changelog、Owner 和关联需求。
- [ ] 固定黄金集、风险切片和 holdout 运行完成，与当前基线比较。
- [ ] 失败样本、波动、成本、TTFT、拒答和安全攻击均分析。
- [ ] 数据分类/区域/供应商/模型能力与 fallback 合法。
- [ ] 灰度分桶稳定，不在同一会话中途切换；回滚版本存在。
- [ ] 发布与回滚写审计，仪表盘可按 config version 分解。

## 8. 知识发布清单

- [ ] 来源、Owner、有效期、密级、许可/使用权和 ACL 已确认。
- [ ] 文件扫描、解析质量、页码/章节、表格、chunk 统计和抽查通过。
- [ ] 新旧版本、上线/下线时间、冲突文档和删除语义明确。
- [ ] 受影响黄金集和 ACL/注入/过期切片通过。
- [ ] Staged 索引完整，原子切换，旧版本与缓存失效可回滚。
- [ ] 知识管理员可看到失败并执行重试/回滚；发布已审计。

## 9. 生产发布 Go/No-Go

### 产品/质量

- [ ] MVP/Production Ready 范围与 UAT 签字完成，P0/P1=0。
- [ ] 检索、忠实度、引用、拒答、安全、性能和成本门禁通过。
- [ ] 已知限制、用户提示、人工升级和反馈处理流程完成。

### 工程/平台

- [ ] Release manifest、image digest/签名、SBOM、迁移和配置版本齐备。
- [ ] staging 使用同一制品完成部署、迁移、smoke、回滚和故障演练。
- [ ] 容量余量、Provider 配额、HPA/PDB、SSE timeout、队列与 DB 参数批准。

### 安全/数据/许可

- [ ] 威胁模型、跨租户/ACL、渗透/红队、secret/log 扫描通过。
- [ ] 数据清单、供应商数据流、保留/删除、审计和事件流程批准。
- [ ] 开源/模型/数据/字体/镜像许可证与商标义务批准。

### 运维

- [ ] SLO/错误预算、仪表盘、告警、Runbook、on-call、供应商联系人启用。
- [ ] PITR/对象/IaC/secret 恢复演练达到 RPO/RTO。
- [ ] 灰度、停止、回滚、关停开关和发布后观察窗口/责任人明确。

任一安全/数据一票否决项、P0/P1、不可恢复迁移或无可用回滚均为 No-Go。

## 10. 运营移交包

- [ ] 产品需求、范围、架构、ADR、OpenAPI、字段字典和数据清单。
- [ ] 源码、构建说明、依赖锁、SBOM、制品/签名、IaC/Helm、迁移。
- [ ] 测试/评测数据与报告、性能/安全/DR/UAT 证据、风险接受项。
- [ ] Prompt/模型/检索/切分/知识版本及发布/回滚历史。
- [ ] 仪表盘、告警、Runbook、值班/升级/事件联系人和供应商 SLA。
- [ ] 预算、容量、配额、密钥/证书轮换、漏洞/升级/EOL 日历。
- [ ] 用户、知识管理员、开发、SRE、安全/审计培训材料和录制/签到。

## 11. 上线后 30/60/90 天

| 时间 | 重点 | 输出 |
|---|---|---|
| 30 天 | 稳定性、真实失败、成本、反馈分类 | 首月运营复盘、门槛校准、P1/P2 清单 |
| 60 天 | 知识新鲜度、组织/ACL、模型路由与容量 | 优化实验、恢复/安全小演练、第二场景建议 |
| 90 天 | 业务收益、TCO、平台/自研路线、下一阶段范围 | 阶段验收、路线图与预算决策 |

业务收益必须与 S0 基线比较：任务时长、首解率/升级率、满意度、知识维护成本；不能以问题数或 token 消耗替代价值。
